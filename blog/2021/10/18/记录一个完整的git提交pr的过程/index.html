记录一个完整的git提交pr的过程

时间： 2021-10-18   |   分类： 运维     |   阅读： 952 字 ~2分钟

原文链接

今天提交了第一个pr，过程中查了很多资料，这里做个总结文字。

git下载

https://git-scm.com/downloads

git配置邮箱和用户名

打开git bash，分别执行以下两句命令git config –global user.name “用户名”git config –global user.email “邮箱”

fork仓库

将想要修改的仓库fork到自己的仓库里

git配置ssh

git不用麻烦的https，而是用ssh，所以要配置下ssh。

随便在哪个文件夹上右键，选择 git bash here，输入：

ssh-keygen -t rsa -C “邮箱”

执行命令后需要进行3次或4次确认：

确认秘钥的保存路径（如果不需要改路径则直接三次回车）；

如果上一步置顶的保存路径下已经有秘钥文件，则需要确认是否覆盖（如果之前的秘钥不再需要则直接回车覆盖，如需要则手动拷贝到其他目录后再覆盖）；

创建密码（如果不需要密码则直接回车）；

确认密码；

在当前路径文件夹下会生成2个名为id_rsa和id_rsa.pub的文件，记事本打开rsa.pub并复制。当然你也可以命令查看：

cat ~/.ssh/id_rsa.pub

然后打开github，右上角头像下选择settings，选择SSH keys，new一个，将刚才复制的的内容粘贴进去即可。

准备工作完成，一次配置以后就不用麻烦了。

clone代码到本地

执行命令：

git clone git@github.com/项目作者/项目名

好了之后你就会发现本地上已经把项目下载下来了。

然后用cd命令进入该项目目录：

cd 项目名

与源码仓库建立连接

这里的仓库是指原作者的仓库。

git remote add upstream 仓库地址.git（例如：https://github.com/apache/flink.git）

随后我们用 git remote -v命令，可以看到除了自己的origin也有原作者的upstream了。

创建分支brench

git checkout -b 自定义分支名

本来项目叫item，你可以取名为item-a

本地改代码

随便用啥编辑器，把代码改好。

改好后可以用 git status 看看modified那些文件被修改了。（假定这里是header.php，下面用到）

add修改的文件

git add header.php

提交到分支

git commit -m “提交说明”

将分支推动到仓库

git push origin 分支名

github查看

登录github，在自己的远程仓库便能看刚才push上去的分支了，点击New pull request，compare处选择刚才提交上来的分支，点Create pull request，

写好名字，写好说明，提交，就OK啦。
